# Judo's Data

![Judo Data Pipeline Banner](judo_banner.jpg)

A Python pipeline to extract, clean, and export data from judo competitions using the public API of the International Judo Federation (IJF).


## Synopsis

I'm a Data Scientist, but I'm still a Judoka!! This project came about from a trigger generated by an ML discipline. I noticed that we had few real data sources we could work with from international competitions. Since I did the data extraction part in a dirty and quick way, I decided to polish this first phase with the knowledge I've gained in recent months.


## Overview

This project performs the ETL (Extract, Transform, Load) process for IJF data:
1. **Extraction:** Fetches information about competitions, matches, and athletes from the public API (`data.ijf.org`).
2. **Transformation:** Cleans and structures the data using `pandas`, generating three main datasets:
    - `judo_atle`: Athlete data (name, country, gender, height, weight, etc.).
    - `judo_compt`: Match/competition data.
    - `judo_atle_compt`: Unified dataset.
3. **Load (Export):** Saves the datasets in popular formats for data analysis (`.csv` and `.parquet`).

## Requirements

- Python 3.13+
- [uv](https://github.com/astral-sh/uv) (Recommended) or `pip` with traditional virtual environment

## Installation

### Clone the Project

```bash
git clone https://github.com/caiosaboia/judo_data.git
```

### Option 1: With `uv` (Recommended)

1. Clone or download this project.
2. Sync dependencies using `uv`:

```powershell
uv sync
```

### Option 2: Without `uv` (Traditional)

1. Clone or download this project.
2. Create and activate a virtual environment:

```powershell
# Windows (PowerShell)
python -m venv .venv
.venv\Scripts\Activate.ps1

# Windows (CMD)
python -m venv .venv
.venv\Scripts\activate.bat

# Linux/macOS
python3 -m venv .venv
source .venv/bin/activate
```

3. Install production dependencies:

```powershell
pip install -r requirements.txt
```

> [!TIP]
> If you want to run tests or contribute to development, install development dependencies instead:
> 
> ```powershell
> pip install -r requirements-dev.txt
> ```

## How to Run

### With `uv`:

To start the pipeline, run:

```powershell
uv run python main.py
```

### Without `uv` (With active virtual environment):

To start the pipeline, run:

```powershell
python main.py
```

When executing the command, type the desired year in the prompt. The pipeline will begin collecting data, applying transformations, and generating files in the `data/` directory.

## Generated Datasets

Data is exported to the `data/` folder in `.csv` and `.parquet` formats.

- `judo_atle`
- `judo_compt`
- `judo_atle_compt`

## How to Run Tests

The project follows TDD (Test-Driven Development) methodology and has tests covering all phases.

### With `uv`:

```powershell
uv run pytest tests/ -v
```

### Without `uv` (With active virtual environment and development dependencies installed):

```powershell
pytest tests/ -v
```

## Technologies
- `httpx`: Asynchronous HTTP client to consume the API.
- `pandas`: Data processing and cleaning.
- `pyarrow`: Engine support for exporting in Parquet format.
- `pytest` / `pytest-asyncio` / `respx`: Unit tests and API mocks.
